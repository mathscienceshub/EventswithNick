Production go-live path

  1. Buy a custom domain such as eventswithnick.co.za.
  2. Create the Supabase database using the included SQL file.
  3. Add the Supabase URL and anon/public key in assets/js/backend-config.js.
  4. Move owner login to Supabase Auth for secure live use.
  5. Deploy the final website and hand over the owner login.

Step 4: Add the values into the website

Open assets/js/backend-config.js and change the file from local mode to Supabase mode.

window.EWN_BACKEND = {
  provider: 'local',
  supabaseUrl: '',
  supabaseAnonKey: ''
};

Change it to this, using the real values from the Supabase project:

window.EWN_BACKEND = {
  provider: 'supabase',
  supabaseUrl: 'https://your-project-id.supabase.co',
  supabaseAnonKey: 'your-anon-public-key-here'
};

Never use the service_role key inside website files. Only use the anon/public key.
